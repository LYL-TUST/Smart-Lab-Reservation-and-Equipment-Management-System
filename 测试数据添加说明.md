# 测试数据添加说明

本文档说明如何为智能实验室预约与设备管理系统添加测试数据。系统提供了两种方法来添加测试数据。

## 方法1：通过前端界面添加（推荐）

这是最推荐的方法，可以直接在前端页面通过界面操作添加数据。

### 前置条件

1. 确保后端服务已启动（默认端口：8080）
2. 确保前端服务已启动（默认端口：5173）
3. 确保已登录系统，并且具有相应的权限：
   - **添加实验室**：需要管理员（ADMIN）权限
   - **添加设备**：需要管理员（ADMIN）权限
   - **添加课程**：需要教师（TEACHER）或管理员（ADMIN）权限

### 添加实验室

1. 登录系统后，进入 **实验室管理** 页面
2. 点击右上角的 **添加实验室** 按钮
3. 填写表单信息：
   - **实验室名称**：必填，如"计算机实验室A"
   - **位置**：必填，如"教学楼A栋3楼301"
   - **容量**：必填，数字，如50
   - **状态**：选择状态（空闲/已预约/使用中/维护中）
   - **描述**：可选，实验室的详细描述
4. 点击 **确定** 按钮提交

### 添加设备

1. 进入 **设备管理** 页面
2. 点击右上角的 **添加设备** 按钮
3. 填写表单信息：
   - **设备名称**：必填，如"高性能计算机"
   - **型号**：必填，如"Dell OptiPlex 7090"
   - **所属实验室**：必填，从下拉列表中选择一个实验室
   - **购买日期**：必填，选择日期
   - **状态**：选择状态（空闲/已预约/使用中/维护中/报废）
   - **描述**：可选，设备的详细描述
4. 点击 **确定** 按钮提交

### 添加课程

1. 进入 **课程管理** 页面
2. 点击右上角的 **添加课程** 按钮
3. 填写表单信息：
   - **课程名称**：必填，如"Java程序设计"
   - **课程代码**：必填，如"CS101"
   - **授课教师**：必填，从下拉列表中选择一个教师（必须是TEACHER角色的用户）
   - **学期**：必填，如"2024-2025学年第一学期"
   - **学生人数**：必填，数字，如45
   - **课程描述**：可选，课程的详细描述
4. 点击 **确定** 按钮提交

### 注意事项

- 添加设备时，必须先有实验室数据，因为设备必须关联到一个实验室
- 添加课程时，必须先有教师用户数据，因为课程必须关联到一个教师
- 如果遇到权限错误，请使用管理员账号登录
- 如果下拉列表为空，请先添加相应的基础数据

## 方法2：在数据库中直接插入数据

如果需要在数据库中直接批量插入测试数据，可以使用提供的SQL脚本。

### 前置条件

1. 确保MySQL数据库已启动
2. 确保数据库 `lab_management` 已创建
3. 确保已执行 `schema.sql` 创建表结构
4. 确保已执行 `init_users.sql` 创建用户数据

### 执行步骤

1. **打开MySQL客户端**（可以使用命令行、Navicat、DBeaver等工具）

2. **连接到数据库**
   ```sql
   mysql -u root -p
   USE lab_management;
   ```

3. **执行测试数据脚本**
   ```sql
   source backend/src/main/resources/test_data.sql;
   ```
   或者直接复制脚本内容到MySQL客户端执行

4. **验证数据**
   ```sql
   -- 检查实验室数量
   SELECT COUNT(*) FROM laboratory;
   
   -- 检查设备数量
   SELECT COUNT(*) FROM equipment;
   
   -- 检查课程数量
   SELECT COUNT(*) FROM course;
   
   -- 查看实验室列表
   SELECT id, name, location, capacity, status FROM laboratory;
   
   -- 查看设备列表
   SELECT id, name, model, lab_id, status FROM equipment;
   
   -- 查看课程列表
   SELECT id, name, code, teacher_id, semester FROM course;
   ```

1. **外键约束**：
   - 设备必须关联到已存在的实验室
   - 课程必须关联到已存在的教师用户
   - 预约必须关联到已存在的用户和实验室
   - 如果遇到外键约束错误，请检查：
     - 用户数据是否已正确插入（执行 `init_users.sql`）
     - 实验室ID是否正确
     - 教师ID是否正确（假设teacher1=2, teacher2=3，admin=1）

2. **ID冲突**：
   - 如果表中已有数据，可能会遇到ID冲突
   - 可以：
     - 删除现有数据后重新插入（谨慎使用）
     - 或者修改脚本中的ID值

3. **密码加密**：
   - 用户密码会在后端启动时自动加密（通过 `DataInitializer`）
   - 如果密码未加密，可以访问 `/api/test/init-passwords` 接口加密所有密码

4. **数据清理**：
   - 脚本开头有清空数据的SQL语句（已注释）
   - 如需清空数据，请谨慎取消注释并执行

## 常见问题

### Q1: 前端添加数据时提示权限不足

**解决方案**：
- 使用管理员账号（admin/admin123）登录
- 检查用户角色是否正确
- 检查后端权限配置

### Q2: 添加设备时下拉列表中没有实验室

**解决方案**：
- 先添加实验室数据
- 刷新页面
- 检查后端API是否正常返回数据

### Q3: 添加课程时下拉列表中没有教师

**解决方案**：
- 确保数据库中有TEACHER角色的用户
- 执行 `init_users.sql` 创建测试教师账号
- 刷新页面

### Q4: SQL脚本执行时遇到外键约束错误

**解决方案**：
1. 检查是否已执行 `init_users.sql`
2. 检查实验室ID是否正确（脚本中假设从1开始）
3. 检查教师ID是否正确（假设teacher1=2, teacher2=3）
4. 可以手动查询ID：
   ```sql
   SELECT id, username, name, role FROM user WHERE role = 'TEACHER';
   SELECT id, name FROM laboratory;
   ```

### Q5: 状态值不匹配

**解决方案**：
- 实验室状态：IDLE（空闲）、RESERVED（已预约）、IN_USE（使用中）、MAINTENANCE（维护中）
- 设备状态：IDLE（空闲）、RESERVED（已预约）、IN_USE（使用中）、MAINTENANCE（维护中）、SCRAPPED（报废）
- 确保前端和后端使用相同的状态值

## 推荐流程

对于新系统，推荐按以下顺序添加数据：

1. **用户数据**：执行 `init_users.sql` 创建用户
2. **实验室数据**：通过前端添加或执行SQL脚本
3. **设备数据**：通过前端添加或执行SQL脚本（需要先有实验室）
4. **课程数据**：通过前端添加或执行SQL脚本（需要先有教师用户）
5. **预约数据**：通过前端添加或执行SQL脚本（需要先有用户和实验室）

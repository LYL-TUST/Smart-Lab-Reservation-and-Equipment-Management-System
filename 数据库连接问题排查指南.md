# 数据库连接问题排查指南

## 错误信息
```
Could not open JDBC Connection for transaction
Communications link failure
The driver has not received any packets from the server
```

这个错误表示后端无法连接到MySQL数据库。

## 排查步骤

### 1. 检查MySQL服务是否运行

#### Windows系统
```bash
# 方法1：打开服务管理器
Win + R，输入 services.msc，回车
找到 "MySQL80" 或 "MySQL" 服务，确保状态为"正在运行"

# 方法2：使用命令行
net start MySQL80
# 或
net start MySQL
```

#### macOS系统
```bash
# 检查MySQL服务状态
brew services list

# 启动MySQL服务
brew services start mysql
```

#### Linux系统
```bash
# 检查MySQL服务状态
sudo systemctl status mysql
# 或
sudo systemctl status mysqld

# 启动MySQL服务
sudo systemctl start mysql
# 或
sudo systemctl start mysqld
```

### 2. 检查数据库配置

打开 `backend/src/main/resources/application.yml`，检查以下配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lab_management?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root          # 修改为您的MySQL用户名
    password: 452325        # 修改为您的MySQL密码
```

**重要检查项：**
- ✅ `localhost` 是否正确（如果是远程数据库，需要改为IP地址）
- ✅ `3306` 端口是否正确（MySQL默认端口）
- ✅ `lab_management` 数据库是否存在
- ✅ `username` 和 `password` 是否正确

### 3. 测试数据库连接

#### 方法1：使用MySQL命令行
```bash
# 登录MySQL
mysql -u root -p

# 输入密码后，执行以下命令检查
SHOW DATABASES;
USE lab_management;
SHOW TABLES;
```

#### 方法2：使用MySQL Workbench
1. 打开MySQL Workbench
2. 创建新连接
3. 输入用户名和密码
4. 测试连接

### 4. 创建数据库（如果不存在）

```sql
-- 登录MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE IF NOT EXISTS lab_management 
DEFAULT CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

-- 验证数据库是否创建成功
SHOW DATABASES;
```

### 5. 检查MySQL用户权限

```sql
-- 登录MySQL
mysql -u root -p

-- 检查用户权限
SELECT user, host FROM mysql.user WHERE user = 'root';

-- 如果root用户只能从localhost连接，确保应用也在同一台机器上
-- 如果需要远程连接，需要授权：
-- GRANT ALL PRIVILEGES ON lab_management.* TO 'root'@'%' IDENTIFIED BY 'your_password';
-- FLUSH PRIVILEGES;
```

### 6. 检查防火墙

#### Windows系统
- 检查Windows防火墙是否阻止了MySQL端口（3306）
- 可以在防火墙设置中添加MySQL例外

#### macOS/Linux系统
```bash
# 检查端口是否开放
sudo netstat -tuln | grep 3306
# 或
sudo lsof -i :3306
```

### 7. 检查MySQL配置文件

#### Windows系统
MySQL配置文件通常在：`C:\ProgramData\MySQL\MySQL Server 8.0\my.ini`

#### macOS系统
配置文件通常在：`/usr/local/mysql/my.cnf` 或 `/etc/my.cnf`

#### Linux系统
配置文件通常在：`/etc/mysql/my.cnf` 或 `/etc/my.cnf`

检查配置文件中是否有：
```ini
[mysqld]
bind-address = 127.0.0.1  # 如果设置为0.0.0.0，允许所有IP连接
port = 3306
```

### 8. 常见问题解决

#### 问题1：MySQL服务未启动
**解决方案：** 启动MySQL服务（见步骤1）

#### 问题2：密码错误
**解决方案：** 
1. 确认 `application.yml` 中的密码是否正确
2. 如果忘记密码，可以重置MySQL root密码

#### 问题3：数据库不存在
**解决方案：** 创建数据库（见步骤4）

#### 问题4：端口被占用
**解决方案：**
```bash
# Windows
netstat -ano | findstr :3306
# 找到占用端口的进程ID，结束进程

# macOS/Linux
lsof -i :3306
kill -9 <PID>
```

#### 问题5：MySQL版本不兼容
**解决方案：** 
- 确保使用MySQL 5.7+ 或 8.0+
- 检查JDBC驱动版本是否兼容

## 快速修复步骤

1. **启动MySQL服务**
   ```bash
   # Windows
   net start MySQL80
   
   # macOS
   brew services start mysql
   
   # Linux
   sudo systemctl start mysql
   ```

2. **验证数据库存在**
   ```sql
   mysql -u root -p
   SHOW DATABASES;
   -- 如果lab_management不存在，创建它
   CREATE DATABASE lab_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. **检查配置文件**
   - 打开 `backend/src/main/resources/application.yml`
   - 确认 `username` 和 `password` 正确
   - 确认 `url` 中的数据库名正确

4. **重启后端服务**
   - 停止后端服务
   - 重新启动后端服务
   - 查看启动日志，确认数据库连接成功

## 验证连接成功

后端启动成功后，日志中应该显示：
```
HikariPool-1 - Starting...
HikariPool-1 - Start completed.
```

如果看到这些信息，说明数据库连接成功。

## 如果问题仍然存在

1. 查看后端完整日志，找到具体的错误信息
2. 检查MySQL错误日志
3. 确认MySQL版本和JDBC驱动版本兼容
4. 尝试使用MySQL Workbench手动连接，验证配置是否正确

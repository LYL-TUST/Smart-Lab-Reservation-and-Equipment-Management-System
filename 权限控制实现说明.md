# 权限控制实现说明

## 问题描述
所有用户（不管是哪个测试账号或自己创建的账号）都显示了管理员的权限，需要实现权限的区分。

## 解决方案

### 1. 后端权限控制

#### 1.1 修复 AuthService.java
**问题**：第59行调用 `user.getRole().name()` 报错，因为 `role` 字段是 `String` 类型，不是枚举类型。

**修复**：
```java
// 修改前
String token = jwtUtil.generateToken(
    user.getId(),
    user.getUsername(),
    user.getRole().name()  // 错误：String 没有 name() 方法
);

// 修改后
String token = jwtUtil.generateToken(
    user.getId(),
    user.getUsername(),
    user.getRole()  // 正确：直接使用 String
);
```

#### 1.2 后端已有的权限控制
后端已经实现了基于注解的权限控制：

- **SecurityConfig.java**：启用了方法级权限控制 `@EnableGlobalMethodSecurity(prePostEnabled = true)`
- **Controller 层**：使用 `@PreAuthorize` 注解控制接口访问权限

示例：
```java
// LaboratoryController.java
@PostMapping
@PreAuthorize("hasRole('ADMIN')")  // 只有管理员可以创建
public Result<Laboratory> create(@Validated @RequestBody LaboratoryDTO dto) { ... }

@PutMapping("/{id}")
@PreAuthorize("hasRole('ADMIN')")  // 只有管理员可以编辑
public Result<Laboratory> update(@PathVariable Long id, @Validated @RequestBody LaboratoryDTO dto) { ... }

// ReservationController.java
@PostMapping("/course")
@PreAuthorize("hasAnyRole('TEACHER', 'ADMIN')")  // 教师和管理员可以创建课程预约
public Result<List<Reservation>> createCourse(...) { ... }

@PutMapping("/{id}/approve")
@PreAuthorize("hasRole('ADMIN')")  // 只有管理员可以审批
public Result<Reservation> approve(...) { ... }
```

### 2. 前端权限控制

#### 2.1 权限配置（已存在）
**文件**：`frontend/src/config/permissions.js`

定义了三种角色及其权限：
- **ADMIN（管理员）**：拥有所有权限
- **TEACHER（教师）**：可以管理实验室、设备、课程，审批预约
- **STUDENT（学生）**：只能查看和预约，管理自己的预约

#### 2.2 权限指令（已存在）
**文件**：`frontend/src/directives/permission.js`

提供了 `v-permission` 指令用于控制页面元素的显示/隐藏。

### 5. 验证方法

1. **使用不同角色登录**：
   - 管理员登录：可以看到所有操作按钮（添加、编辑、删除等）
   - 教师登录：可以看到部分操作按钮（编辑、维护等），但看不到删除按钮
   - 学生登录：只能看到查看按钮，看不到任何管理操作按钮

2. **检查菜单显示**：
   - 管理员：可以看到"用户管理"菜单
   - 教师/学生：看不到"用户管理"菜单

3. **测试后端接口**：
   - 即使前端绕过权限控制，后端也会拦截未授权的请求
   - 返回 403 Forbidden 错误

### 6. 权限控制流程

```
用户登录
  ↓
后端验证用户名密码
  ↓
生成 JWT Token（包含用户角色）
  ↓
前端存储 Token 和用户信息
  ↓
前端根据用户角色显示/隐藏按钮
  ↓
用户点击操作按钮
  ↓
前端发送请求（携带 Token）
  ↓
后端验证 Token 和权限
  ↓
执行操作或返回 403 错误
```

### 7. 注意事项

1. **前端权限控制**：主要用于优化用户体验，隐藏用户无权访问的功能
2. **后端权限控制**：是真正的安全保障，防止恶意请求
3. **双重验证**：前后端都需要进行权限验证，确保系统安全

## 总结

通过以上修改，系统已经实现了完整的权限控制：
- ✅ 修复了后端编译错误（AuthService.java 第59行）
- ✅ 前端页面根据用户角色显示不同的操作按钮
- ✅ 后端接口通过注解控制访问权限
- ✅ 路由守卫控制页面访问权限
- ✅ 菜单根据用户角色动态显示

现在不同角色的用户登录后，将看到与其权限相匹配的界面和功能。

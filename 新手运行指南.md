# 智能实验室预约与设备管理系统 - 新手运行指南

> 💡 **快速开始（5分钟）**
> 
> 1. 安装 JDK 1.8+、Maven、MySQL 5.7+、Node.js 16+
> 2. 创建数据库：`CREATE DATABASE lab_management;`
> 3. 导入表结构：`mysql -u root -p lab_management < backend/src/main/resources/schema.sql`
> 4. 导入用户数据：`mysql -u root -p lab_management < backend/src/main/resources/init_users.sql`
> 5. 修改 `backend/src/main/resources/application.yml` 中的数据库密码
> 6. 运行后端：`cd backend && mvn spring-boot:run`
> 7. 运行前端：`cd frontend && npm install && npm run dev`
> 8. 访问：http://localhost:5173，使用 `admin` / `admin123` 登录
>
> 📖 **详细步骤请查看下方文档**

## 📖 项目介绍

智能实验室预约与设备管理系统是一个基于 Spring Boot + Vue 3 的实验室管理系统，支持：

- ✅ 实验室管理（查看、添加、编辑、删除）
- ✅ 设备管理（查看、添加、编辑、删除、维护）
- ✅ 预约管理（创建预约、审批预约、查看预约）
- ✅ 课程管理（创建课程、绑定预约）
- ✅ 用户管理（管理员功能）
- ✅ 权限控制（管理员、教师、学生三种角色）

**技术栈：**
- 后端：Spring Boot 2.7.18 + MyBatis Plus + MySQL + JWT
- 前端：Vue 3 + Vite + Element Plus + Pinia + Vue Router

## 💾 软件下载与安装

### 1. JDK 安装

#### Windows 系统

1. **下载 JDK**
   - 访问 Oracle 官网：https://www.oracle.com/java/technologies/downloads/
   - 或访问 OpenJDK：https://adoptium.net/
   - 下载 JDK 1.8 或更高版本（推荐 JDK 8 或 JDK 11）

2. **安装 JDK**
   - 运行下载的安装程序
   - 按照提示完成安装（建议安装到默认路径）

3. **配置环境变量**
   - 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"
   - 在"系统变量"中新建：
     - 变量名：`JAVA_HOME`
     - 变量值：JDK安装路径（如：`C:\Program Files\Java\jdk1.8.0_xxx`）
   - 编辑"Path"变量，添加：`%JAVA_HOME%\bin`

4. **验证安装**
   ```bash
   java -version
   javac -version
   ```
   如果显示版本信息，说明安装成功。

#### macOS 系统

```bash
# 使用 Homebrew 安装
brew install openjdk@8

# 或下载安装包
# 访问 https://adoptium.net/ 下载 macOS 版本
```

#### Linux 系统

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-8-jdk

# CentOS/RHEL
sudo yum install java-1.8.0-openjdk-devel
```

### 2. Maven 安装

#### Windows 系统

1. **下载 Maven**
   - 访问：https://maven.apache.org/download.cgi
   - 下载 `apache-maven-3.x.x-bin.zip`

2. **解压安装**
   - 解压到任意目录（如：`C:\Program Files\Apache\maven`）

3. **配置环境变量**
   - 新建系统变量：
     - 变量名：`MAVEN_HOME`
     - 变量值：Maven解压路径（如：`C:\Program Files\Apache\maven`）
   - 编辑"Path"变量，添加：`%MAVEN_HOME%\bin`

4. **验证安装**
   ```bash
   mvn -version
   ```

#### macOS/Linux 系统

```bash
# macOS (使用 Homebrew)
brew install maven

# Linux (Ubuntu/Debian)
sudo apt install maven

# 验证
mvn -version
```

### 3. MySQL 安装

#### Windows 系统

1. **下载 MySQL**
   - 访问：https://dev.mysql.com/downloads/mysql/
   - 选择 Windows 版本，下载 MySQL Installer

2. **安装 MySQL**
   - 运行安装程序
   - 选择"Developer Default"或"Server only"
   - 设置 root 用户密码（**请记住这个密码，后续配置需要**）
   - 完成安装

3. **启动 MySQL 服务**
   - 打开"服务"（Win+R，输入 `services.msc`）
   - 找到"MySQL80"服务，确保状态为"正在运行"

4. **验证安装**
   ```bash
   mysql -u root -p
   ```
   输入密码，如果进入 MySQL 命令行，说明安装成功。

#### macOS 系统

```bash
# 使用 Homebrew 安装
brew install mysql

# 启动 MySQL
brew services start mysql

# 设置 root 密码
mysql_secure_installation
```

#### Linux 系统

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server

# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 设置 root 密码
sudo mysql_secure_installation
```

## 🗄️ 数据库配置

### 1. 创建数据库

打开 MySQL 命令行或 MySQL Workbench，执行以下命令：

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS lab_management 
DEFAULT CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE lab_management;
```

### 2. 导入数据库结构

在项目根目录下，找到 `backend/src/main/resources/schema.sql` 文件，执行：

**方法1：使用 MySQL 命令行**
```bash
# 进入项目目录
cd backend/src/main/resources

# 执行 SQL 文件
mysql -u root -p lab_management < schema.sql
```

**方法2：使用 MySQL Workbench**
1. 打开 MySQL Workbench
2. 连接到数据库
3. 选择 `lab_management` 数据库
4. 点击"File" → "Open SQL Script"
5. 选择 `backend/src/main/resources/schema.sql`
6. 点击执行按钮（⚡）

**方法3：在 MySQL 命令行中执行**
```sql
-- 登录 MySQL
mysql -u root -p

-- 使用数据库
USE lab_management;

-- 执行 SQL 文件
source E:/总文件夹/前端作品集/智能实验室预约与设备管理系统/backend/src/main/resources/schema.sql;
```

### 3. 导入初始用户数据

执行用户初始化脚本：

```bash
# 方法1：命令行
mysql -u root -p lab_management < backend/src/main/resources/init_users.sql

# 方法2：MySQL 命令行
source backend/src/main/resources/init_users.sql;
```

**默认管理员账号：**
- 用户名：`admin`
- 密码：`admin123`

### 4. 导入测试数据（可选）

如果需要测试数据，可以执行：

```bash
# 导入测试数据
mysql -u root -p lab_management < backend/src/main/resources/test_data.sql

# 导入设备数据（如果 test_data.sql 中没有设备数据）
mysql -u root -p lab_management < 插入设备测试数据.sql
```

**注意：** 如果遇到中文编码问题，请先执行：
```sql
SET NAMES utf8mb4;
SET CHARACTER SET utf8mb4;
```

### 5. 验证数据库

```sql
-- 查看所有表
SHOW TABLES;

-- 查看用户数据
SELECT id, username, name, role FROM user WHERE deleted = 0;

-- 查看实验室数据
SELECT id, name, location, status FROM laboratory WHERE deleted = 0;
```

## ⚙️ 后端配置与运行

### 1. 修改数据库配置

打开 `backend/src/main/resources/application.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lab_management?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root          # 修改为您的 MySQL 用户名
    password: 452325        # 修改为您的 MySQL 密码
```

**重要：** 请将 `username` 和 `password` 修改为您实际的 MySQL 账号密码！

### 2. 安装依赖

在项目根目录下，进入 `backend` 文件夹：

```bash
cd backend

# Maven 会自动下载依赖，首次运行可能需要几分钟
# 如果下载慢，可以配置国内镜像（见常见问题）
```

### 3. 运行后端

**方法1：使用 IDE（推荐）**
- 使用 IntelliJ IDEA 或 Eclipse
- 打开项目，找到 `LabManagementApplication.java`
- 右键 → "Run" 或点击运行按钮

**方法2：使用 Maven 命令**
```bash
cd backend
mvn spring-boot:run
```

**方法3：打包后运行**
```bash
cd backend
mvn clean package
java -jar target/laboratory-management-1.0.0.jar
```

### 4. 验证后端运行

后端启动成功后，您会看到类似以下信息：

```
Started LabManagementApplication in X.XXX seconds
```

访问测试接口：
- 浏览器打开：http://localhost:8080/api/auth/test
- 如果返回 `{"code":200,"message":"API正常运行","data":"API正常运行"}`，说明后端运行成功！

**后端默认端口：** `8080`
**API 基础路径：** `http://localhost:8080/api`

## 🐛 常见问题

### 1. Maven 下载依赖慢

**解决方案：** 配置国内镜像

编辑 `~/.m2/settings.xml`（Windows 在 `C:\Users\用户名\.m2\settings.xml`），添加：

```xml
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <name>Aliyun Maven</name>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### 2. npm 安装依赖慢

**解决方案：** 使用国内镜像

```bash
# 设置淘宝镜像
npm config set registry https://registry.npmmirror.com

# 或使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### 3. MySQL 连接失败

**错误信息：** `Access denied for user 'root'@'localhost'`

**解决方案：**
1. 检查 `application.yml` 中的用户名和密码是否正确
2. 确认 MySQL 服务是否启动
3. 检查 MySQL 用户权限

```sql
-- 检查用户权限
SELECT user, host FROM mysql.user;

-- 如果 root 用户不存在，创建用户
CREATE USER 'root'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

### 4. 端口被占用

**错误信息：** `Port 8080 is already in use`

**解决方案：**

**Windows：**
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 结束进程（PID 是上面命令显示的进程ID）
taskkill /PID <PID> /F
```

**macOS/Linux：**
```bash
# 查找占用端口的进程
lsof -i :8080

# 结束进程
kill -9 <PID>
```

或修改 `application.yml` 中的端口：
```yaml
server:
  port: 8081  # 修改为其他端口
```

### 5. 前端无法连接后端

**错误信息：** `Network Error` 或 `CORS error`

**解决方案：**
1. 确认后端是否正常运行
2. 检查后端地址是否正确（`frontend/src/api/request.js`）
3. 确认后端 CORS 配置是否正确（已默认配置允许所有来源）

### 6. 数据库字符编码问题

**错误信息：** `Incorrect string value: '\xAE\xA4A' for column 'name'`

**解决方案：**
```sql
-- 确保数据库使用 utf8mb4
ALTER DATABASE lab_management CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 执行 SQL 前设置编码
SET NAMES utf8mb4;
SET CHARACTER SET utf8mb4;
```

### 7. Java 版本不匹配

**错误信息：** `UnsupportedClassVersionError`

**解决方案：**
- 确保使用 JDK 1.8 或更高版本
- 检查 IDE 中的 JDK 配置
- 检查环境变量 `JAVA_HOME`

### 8. 前端页面空白

**解决方案：**
1. 打开浏览器开发者工具（F12），查看 Console 错误
2. 检查后端 API 是否正常响应
3. 清除浏览器缓存，重新加载

### 9. 登录后跳转失败

**解决方案：**
1. 检查浏览器控制台是否有错误
2. 确认 JWT Token 是否正确存储
3. 检查路由配置是否正确

### 10. 数据库表不存在

**错误信息：** `Table 'lab_management.user' doesn't exist`

**解决方案：**
1. 确认已执行 `schema.sql` 创建表结构
2. 确认使用的是正确的数据库（`lab_management`）
3. 检查表是否被删除，重新执行 `schema.sql`
